# AI Resume Builder

An ASP.NET MVC web application that generates professional, ATS-friendly resumes using Google's Gemini LLM API. Users provide their personal details, skills, experience, and target job role, and the app returns a tailored resume generated by AI.

## Features

- Simple web form to collect user details (name, contact info, skills, experience, education, target job role)
- Integrates with the Gemini API (`gemini-2.5-flash` model) to generate a professional, ATS-optimized resume
- Displays the generated resume directly in the browser
- Built with ASP.NET Core MVC (.NET 8)

## Tech Stack

- **Framework:** ASP.NET Core MVC (.NET 8)
- **Language:** C#
- **AI Model:** Google Gemini (`gemini-2.5-flash`)
- **Frontend:** Razor Views, Bootstrap, jQuery

## Prerequisites

- [.NET 8 SDK](https://dotnet.microsoft.com/download)
- Visual Studio 2022 (or later) / VS Code
- A [Google Gemini API key](https://ai.google.dev/)

## Setup Instructions

1. **Clone the repository**
   ```
   git clone https://github.com/irfan7-git/ResumeBuilderAIAgent.git
   cd ResumeBuilderAIAgent
   ```

2. **Configure your Gemini API key using .NET User Secrets**

   This project reads the API key from configuration rather than hardcoding it. Set it up locally with:
   ```
   dotnet user-secrets init
   dotnet user-secrets set "Gemini:ApiKey" "YOUR_GEMINI_API_KEY_HERE"
   ```

3. **Restore dependencies and run**
   ```
   dotnet restore
   dotnet run
   ```

4. Open the app in your browser at the URL shown in the terminal (e.g. `https://localhost:5001`).

## How It Works

1. The user fills out a form with their name, contact details, skills, work experience, education, and target job role.
2. The `ResumeController` builds a prompt from this data and sends it to the Gemini API's `generateContent` endpoint.
3. Gemini returns a generated resume tailored to the provided details and job role.
4. The resume is displayed back to the user on the results page.

## Security Notes

- API keys are **never hardcoded** in source code. They are managed via [.NET User Secrets](https://learn.microsoft.com/en-us/aspnet/core/security/app-secrets) for local development.
- For production deployments, use environment variables or a secrets manager (e.g. Azure Key Vault) instead of committing keys to any config file.
- `.gitignore` is configured to exclude build artifacts (`bin/`, `obj/`) and any local secret/config files.
